# AgSwarm Client v0.2.22

This release fixes the Windows installer failure seen in v0.2.21.

## Fixed

- Packages the bundled Ag/pi-web runtime as `runtime-node.zip` instead of asking NSIS to install the entire `node_modules` tree file-by-file.
- Extracts the bundled runtime into the app data directory before starting Ag, avoiding Windows path-length and deep dependency write failures during installation.
- Keeps release validation strict by requiring the runtime archive during sidecar checks.
- Uses a project-local npm cache while preparing sidecars so a damaged user npm cache cannot break runtime preparation.

## Verification

- `npm run lint`
- `npm run build`
- `npm run check:sidecars -- --target aarch64-apple-darwin`
- `cargo check`
